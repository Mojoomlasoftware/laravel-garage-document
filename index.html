A) Introduction

Garage Management System for Laravel the ideal way to manage complete Garage Management System. It has different user roles like Admin, Customer, Employee, Support Staff and Accountant.

This user guide covers all options that this laravel provides. Please read the documentation carefully as it probably contains answers to most of the questions that may arise. If you feel some aspect is not covered or have some questions, contact us via e-mail sales@mojoomla.com. We will do our best to answer your in the shortest time.

1). FTP Upload
  1. Unzip the " Garage-package.zip " file.
  2. Upload the extracted "Garage" folder into your server.
  3. Then open Phpmyadmin in your server and create new database. Create database in cPanel Click here
  4. Then open your link in web browser and you will show Installation wizard.

    C) Access Right Settings

    Click on Setting inside Click on Access Right Setting to start setting up the system. Access rights is a functionality for all users like Customer, Support Staff, Employee and Accountant to give access rights for use modules of this system. Here all modules like Inventory, Users, Vehicles, Service etc. in this modules which module with access to add, edit, view and delete access right for all users. So basically admin can give rights to all users for access right to operate this all modules.




      Access Right Settings
    • Accordion with user name: Customer, Employee, Support Staff and Accountant.
    • Module Name: Modules like Vehicle, Supplier, Product, Service etc. This all module have data and this data with make operations like, Add, Edit, Delete, View also Owndata.
    • View: For modules with rights to view this data.
    • Add: For modules with rights to add this data.
    • Update: For modules with rights to update this data.
    • Delete: For modules with rights to delete this data.
    • Owndata: For modules with rights to make operation like edit, and view data, this is owndata rights.

    E) Branch Settings

    Click on Setting inside Click on Branch Setting tab to start setting up the system. You are advised to Admin can update it's current branch from here Branch Setting tab, select Branch name and click on update button after it Admin can move on selected Branch. All the details and data also like Support Staff, Employee, Service, Quotation, Invoice and Reports all the modules data show on this system is only current branch, which is selected by Admin.

      Basic Steps For Use Of Branch Feature:
    • 1) Firstly install successfuly fresh Garage Package, after that Branch Module have not show any data of branch because this is a Main Branch, so this branch name and details are not show on Branch Module. If you need to show Main Branch details then you can go on Settings and click on General Setting tab.
    • 2) If Admin need to create another branch at that time Admin go to Branch Module and click on add branch and fill all required data and submit it. Now successfuly create branch then Admin can see details of this branch on Branch Module listing page, also Admin can add, edit, view and delete this branch from Branch Module.
    • 3) Now inside Branch Module have show one branch entry, so inside Branch Setting Tab Admin can show two branch name on Branch Select Box like (Main Branch and New branch name).
    • 4) Now on Branch Setting Tab Admin can select branch name and update branch so Admin is currently active for this particular selected branch. So now Admin show, add, edit, view and delete data of this current branch, which is selected by Admin.
    • 5) Admin is on Current active branch at a time add one Support Staff, so this Supports Staff is belongs to only for this branch not for any other branch. This Support Staff is able to do or perform any kind of action for only his/her branch based on give permissions by Admin.
    • 6) All roles and data added by Admin is for Admin current selected branch. But in Customer and Supllier case both are common for all the branch, this both role have not any single branch. This is common for all branch. And for customer side any customer login on Garage System at a time customer can show all the details of his/her all branch data like sale, service, quotation and other data.
    • 7) Admin or Support Staff add any kind of data from any modules at that time he/she shown one field of selected branch name, to just confirm this data is add for this selected branch, because this person is for add data is it currently/active on this particular branch and add this data for this particular branch.
